I developed this project to explore the technical challenges of real time object interaction and physics based destruction within the Unity engine. The goal was to recreate the iconic fruit slashing mechanic by using 3D spatial triggers and velocity tracking. I wanted to focus on how a game can feel fluid and satisfying by balancing fast paced player input with accurate physics responses. This project served as a deep dive into the Unity 3D engine, specifically focusing on the interaction between moving colliders and dynamic game objects.

The core of the gameplay is built around a "Blade" system that tracks the mouse movement in 3D space. I implemented a script that converts the 2D screen coordinates of the mouse into a 3D path, creating a trail of colliders that act as the cutting edge. When a fruit object enters this trigger zone at a high enough velocity, the game triggers a "slice" event. Instead of just deleting the fruit, I worked on a system that swaps the whole fruit model with two "sliced" halves and applies an outward force to them, creating the physical explosion effect that makes the game feel impactful.

I created a dynamic spawning system to keep the gameplay continuous and challenging. Using a "Spawner" script, I managed the frequency and trajectory of the fruit being launched into the air, using randomized force and torque to ensure that no two tosses are exactly the same. One of the biggest technical hurdles was managing "object pooling" or cleanup; I had to ensure that the sliced fruit pieces and missed fruits were properly destroyed once they fell out of the camera view to prevent memory leaks and keep the game running at a high frame rate. This taught me the importance of performance optimization in physics heavy games.

Building this game gave me a much stronger grasp of the Unity coordinate system and the importance of "game feel." I spent a lot of time tweaking the trail renderer and the speed requirements for a slash to ensure that the game felt responsive rather than clunky. In the future, I plan to add a combo system that rewards players for slicing multiple fruits in a single swipe and a "bomb" hazard that subtracts points if touched. I am also interested in adding particle effects like juice splashes to further enhance the visual feedback. This project was a major milestone in my 3D development journey, as it pushed me to think about how players interact with objects in motion.
